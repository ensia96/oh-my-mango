# 플랜/빌드 망고 추가 및 메인 망고 위임 구조 변경

## 배경
- 현재: 이슈 망고(이슈+plan.md), PR 망고(브랜치+PR+작업)로 역할 분담
- 문제: 작업 단위와 문서 관리의 경계가 모호함
- 참고: 기존 이슈/PR 망고는 #4 (에이전트 재귀 호출 검증) 테스트용으로 보존

## 작업
- [ ] 플랜 망고 시스템 프롬프트 작성 (신규)
- [ ] 빌드 망고 시스템 프롬프트 작성 (신규)
- [ ] 메인 망고 프롬프트에서 서브에이전트 위임 대상 변경
  - 기존: 이슈 망고, PR 망고
  - 변경: 플랜 망고, 빌드 망고
- [ ] 기존 이슈/PR 망고 파일은 유지 (삭제하지 않음)

## 검증
- 플랜 망고로 이슈/계획문서/브랜치/PR 검토/생성/보고 가능 확인
- 빌드 망고로 작업 진행/검증/보고/기록 가능 확인
- 메인 망고에서 플랜/빌드 망고로 위임 정상 동작 확인
