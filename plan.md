# 메인 망고 오케스트레이션 전용화

## 배경
- 메인 망고가 이슈/PR 등을 직접 조작 시 사이드 이펙트 발생 위험
- 역할 분리로 안정성 확보 필요

## 작업
- [ ] 메인 망고 시스템 프롬프트에 수동 작업 금지 정책 추가
- [ ] 서브 에이전트 위임 로직 명시
- [ ] 예외 처리 플로우 정의 (적합한 서브 에이전트 없음 → 사용자 검토 요청 → 허가 시 실행)
- [ ] 오케스트레이션 전용 역할 명확화

## 검증
- 메인 망고에게 이슈 생성 요청 시 직접 실행하지 않고 서브 에이전트에게 위임하는지 확인
- 적합한 서브 에이전트 없을 때 사용자에게 검토 요청하는지 확인
